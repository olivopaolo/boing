Boing: input events tools

Documentation

Redirecting input sources to different outputs:

> python test/run.py [-i <source>]... [-o <output>]...
   
Options:
  -i <source>
     Define an input source

  -o <output>
     Define an output

  -h, --help
     Show help

Sources:
  TUIO socket URL
     Grab a TUIO stream over a socket 
      e.g. tuio://sim@localhost:3333?rt=False

  TUIO log URL
     Play a TUIO stream logged in a file 
      e.g. tuio:///tmp/test.osc.bz2?speed=1&loop=False&rt=False

     If "rt" (i.e. reception time) is true, the event's timetag is the
     time when the event is received, otherwise the OSC bundle
     timestamp is considered.

  MTDEV device URL  
     Open input device using libmtdev (LINUX ONLY)
      e.g. mtdev:///dev/input/event5

Outputs:
  viz:
     Display gesture events (default output) 
      e.g. viz:?req=(diff,.*,gestures),timetag&hz=60&x=320&y=240

  stat:
     Print data production statistics.
      e.g. stat:?req=.*&clear=False&hz=1

  dump:
     Print the received events to the standard output 
      e.g. dump:?req=.*&hz=None&src=False&dest=False&count=False

  TUIO socket URL
     Redirect gesture events as a TUIO stream to a socket
      e.g. tuio://[::1]:3333?req=(diff,.*,gestures),timetag,osc,data

  TUIO log URL
     Log gesture events as a TUIO log file
      e.g. tuio:///tmp/test.osc.bz2?req=(diff,.*,gestures),timetag,osc,data
