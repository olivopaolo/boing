Boing: input events tools

Documentation

Redirecting input sources to different outputs:

> python test/run.py [-i <source>]... [-o <output>]...
   
Options:
  -i <source>
     Define an input source

  -o <output>
     Define an output

  -h, --help
     Show help

Sources:
  TUIO socket URL
     Grab a TUIO stream over a socket 
     (e.g. tuio://sim@localhost:3333?rt=false)

  TUIO log URL
     Play a TUIO stream logged in a file 
     (e.g. tuio:///tmp/test.osc.bz2?speed=1&loop=false&rt=false)

     The "rt" parameter (i.e. reception time) defines how events'
     timetag is determined.  If true the timetag is determined as the
     moment when the event is received, instead of using the moment
     when the event was generated.

Outputs:
  viz:
     Display gesture events (default output) 
     (e.g. viz:?hz=60&x=320&y=240)

  dump:
     Print the received events to the standard output 
     (e.g. dump:?hz=60)
