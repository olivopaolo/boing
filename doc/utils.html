Boing: Generic utilities examples

Examples:

Dump to console all the available products received from the inputs:

boing -i bridge: -o dump:

Dump requested1 products to a TCP socket:

boing -i bridge: -o dump.tcp://[::1]:7777?request=data

Dump only the products that match a query filter:

boing -i bridge: -o dump:?filter=str

Print statistics (i.e. lag, frequence, sum, tags, etc.) of the products from two sources:

boing -i udp://:0 tcp://:0  -o stat:

Record to file the incoming events:

boing -i bridge: -o log:///tmp/log.bz2

Replay a log file a forward it to a socket:

boing -i play:///tmp/log.bz2 -o udp://[::1]:7777

Add a lag (1000 ms) to product stream:

boing -i bridge:?post=lag:1000 -o bridge:

1The request of a node defines what the node requires to its inputs to produce; it does not means that the only products a node will receive are the requested ones. For example if a product is a necessary step to made the requested product, then also the intermediate product will be normally dispatched. Filtering can be used to exclude unwanted products.
