Boing: URI guide

Platform
L = LinuxW = Windows1
Mode
I = InputO = OutputF = Function

IO devicesURL query keys2
LW I stdin:   Standard input
LW O stdout:   Standard output
LW I file://<absolute-filepath> uncompress, postend Absolute filepath (Read Only)
LW O file://<absolute-filepath>   Absolute filepath (Write Only)
LW I <relative-filepath> uncompress, postend Relative filepath (Read Only)3
LW O <relative-filepath>   Relative filepath (Write Only)3
LW I udp://<host>:<port>   Read from UDP socket
LW O udp://<host>:<port> writeend Write to UDP socket
LW IO tcp://<host>:<port>   Read/Write on TCP socket
Encodings
LW IO slip.<IOdevice>   SLIP encoding/decoding
LW IO osc[.slip].<IOdevice> IO device's query keys, rt OSC encoding/decoding
LW O osc://<absolute-filepath> raw Record OSC log file
LW I osc://<absolute-filepath> loop, speed Play OSC log file
LW IO tuio[.slip].<IOdevice> IO device's query keys, rt multi-touch event vs. TUIO encoding/decoding
LW O tuio://<absolute-filepath> raw Record TUIO log file from multi-touch event encoding
LW I tuio://<absolute-filepath> loop, speed Play TUIO log file and apply multi-touch event decoding
L I mtdev://<absolute-filepath>   Multi-touch event decoding from mtdev device
Utils
LW O dump[.<OutputDevice>]: src, dest, depth, request, hz Dump products to an output device (default stdout)
LW O stat[.<OutputDevice>]: request, hz Print products statistics to an output device (default stdout)
LW O viz: antialiasing, fps, request Display multi-touch contacts
Functions
LW F filter:[<query>] request, hz Discard the products that do not match 'query'
LW F filterout:[<query>] hz Discard the products that match 'query'
LW F lag:[<msec>] request Add a lag to each received product
LW F timekeeper:   Mark each received product with a timetag
LW F calib: matrix, screen, attr, args, request, resultmode Apply a matrix transformation to the requested arguments
(default $diff.added,updated.contacts..rel_pos,rel_speed)
LW F libfilter: uri, attr, args, request, resultmode Filter product data using libfilter

Host
LW I empty same as any address IPv6
LW I 0.0.0.0 IPv4 any address
LW I [::] IPv6 any address
LW IO 127.0.0.1 IPv4 loopback
LW IO [::1] IPv6 loopback
LW IO x.x.x.x specific IPv4 address
LW IO [x:x:x:x:x:x:x:x] specific IPv6 address
LW IO hostname specific hostname

1On Windows, in order to define a file using the scheme file it is necessary to place the character '/' (slash) before the drive letter.
(e.g. file:///C:/Windows/explorer.exe)

2All input nodes also have the query key pre and all output nodes also have the query key post.

3It cannot be used to form composed URLs.
