Boing: URLs guide

L = Linux, W = Windows

TCP/UDP socket
LW udp://[<host>]:<port> UDP socket
LW tcp://[<host>]:<port> TCP socket
Host
LW empty same as any address
LW 0.0.0.0 IPv4 any address
LW [::] IPv6 any address
LW 127.0.0.1 IPv4 loopback
LW [::1] IPv6 loopback
LW x.x.x.x specific IPv4 address
LW [x:x:x:x:x:x:x:x] specific IPv6 address
LW hostname specific hostname
File1
LW file://<absolute-filepath>
LW <absolute-filepath>
LW <relative-filepath>
SLIP
LW slip.tcp://[<host>]:<port> SLIP encoding/decoding over TCP socket
LW slip.file://<absolute-filepath> SLIP data encoding/decoding over file
OSC
LW osc[.udp|tcp]://[<host>]:<port> OSC through a socket (SLIP decoding for TCP connections)
LW osc://<absolute-filepath> OSC from file (SLIP decoding)
TUIO
LW tuio[.udp|tcp]://[<host>]:<port> TUIO/OSC through a socket (SLIP decoding for TCP connections)
LW tuio://<absolute-filepath> TUIO/OSC from file (SLIP decoding)

1On Windows, in order to define a file using the scheme file it is necessary to place the character '/' (slash) before the drive letter.
(e.g. file:///C:/Windows/explorer.exe)
