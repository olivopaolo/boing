Boing: URI guide

Availability
L = LinuxW = Windows1
I = InputO = Output

Utils
LW I stdin: Read data from standard input
LW O stdout: Write data to standard output
LW O dump: Dump products to standard output
LW O stat: Print products statistics to standard output
Bridges
LW IO udp|tcp://<host>:<port> UDP/TCP socket
LW IO [file://]<absolute-filepath> Absolute filepath
LW IO <relative-filepath> Relative filepath
LW IO slip.udp|tcp://<host>:<port> SLIP data encoding/decoding over UDP/TCP socket
LW IO slip.file://<absolute-filepath> SLIP data encoding/decoding over file
LW I osc[[.slip].udp|tcp]://<host>:<port> OSC through a socket
LW I osc[[.slip].file]://<absolute-filepath> OSC from file (SLIP encoding if empty)
LW I tuio://<host>:<port> TUIO events from socket
Record/Replay
LW O log[.osc]://<absolute-filepath> Log data into OSC packets and write to file
LW I play.osc://<absolute-filepath> Replay OSC from file
LW I play.tuio://<absolute-filepath> Replay TUIO from file
Gestural input support
L I mtdev://<absolute-filepath> Open input device using libmtdev
LW O viz: Display multi-touch contacts

Host
LW I empty same as any address IPv6
LW I 0.0.0.0 IPv4 any address
LW I [::] IPv6 any address
LW IO 127.0.0.1 IPv4 loopback
LW IO [::1] IPv6 loopback
LW IO x.x.x.x specific IPv4 address
LW IO [x:x:x:x:x:x:x:x] specific IPv6 address
LW IO hostname specific hostname

1On Windows, in order to define a file using the scheme file it is necessary to place the character '/' (slash) before the drive letter.
(e.g. file:///C:/Windows/explorer.exe)
