Boing: TUIO examples

General documentation on TUIO can be found at http://www.tuio.org/.

All the proposed software works with the specification TUIO 1.1.

Examples:

Show multi-touch events of a TUIO stream received on a UDP socket (default port 3333):

> boing -i tuio: -o viz:

Record to a file a TUIO stream received on a UDP socket:

> boing -i tuio://:3334 -o osc:///tmp/log.osc.bz2

Replay a TUIO log to an UDP socket:

> boing -i tuio:///tmp/log.osc.bz2 -o udp://[::1]:7777

Replay a TUIO log (at double speed and loop enabled) to a TCP socket and show it locally (antialised at 30 fps):

> boing -i "tuio:///tmp/log.osc.bz2?speed=2&loop" -o slip.tcp://[::1]:7777 "viz:?antialiasing&fps=30"

Convert mtdev events to TUIO and send them through a socket (administrator privileges may be required to read from device node):

> boing -i mtdev:///dev/input/event5 -o tuio://[::1]:3333

Merge a TUIO stream and a mtdev device into a single TUIO stream and print it on the stdout:

> boing -i mtdev:///dev/input/event5 tuio: -o tuio.stdout:

1TUIO packets read/written on TCP sockets or files are supposed to be SLIP encoded/decoded.
